CodeIgniterAdminPro

CodeIgniterAdminPro is a readymade web application that allows creating an administrative panel with CodeIgniter framework within minutes.

Files Details & Functionality

  1. application/config/

    1. application/config/autoload.php

      In this file, we've specified some mostly used libraries (database, session) and helpers (url, file, form, user_email, common) to loaded by default.

    2. application/config/config.php

      Some basic configuration veriables are specified in this file. (base_url, public_url, upload_url, upload_path, db_hostname, db_username, db_password, and db_name)

    3. application/config/routes.php

      In this file, we've defined default controller and some routing rules.

  2. application/controllers/

    1. application/controllers/admin/Administrative.php

      This controller controls all the Admin Management functionalities. All methods of the Administrative class and their functionality are given below.

      • __construct(): Loads the required library, model, and helper. Also, assign some variables for using in the other methods.
      • index(): Loads the right method based on the admin logged-in status.
      • login(): Admin account login.
      • forgotPassword(): Admin account forgot password.
      • resetPassword(): Admin account reset new password.
      • logout(): Logout admin from administrative panel.
      • dashboard(): Admin dashboard.
      • profile(): Admin account information update.
      • oldpass_check(): Validation callback function used to check the old password.
      • email_check(): Validation callback function used to check existing email.
      • username_check(): Validation callback function used to check existing username.
      • forgot_email_check(): Validation callback function used to check existing email on forgot password request.
      • file_check(): Validation callback function used to check file types.

    2. application/controllers/admin/Manage_users.php

      This controller controls all the User Management functionalities. All methods of Manage_users class and their functionality are given below.

      • __construct(): Loads the required library, model, and helper. Also, assign some variables for using in the other methods.
      • index(): The users list with add, edit, delete, and details link.
      • add(): Add new user information.
      • edit(): Update user information.
      • details(): User profile details.
      • delete(): Delete user profile data.
      • block(): Update user account status to inactive.
      • unblock(): Update user account status to active.
      • email_check(): Validation callback function, used to check existing email.
      • forgot_email_check(): Validation callback function, used to check existing email on forgot password request.
      • oldpass_check(): Validation callback function, used to check old password.
      • file_check(): Validation callback function, used to check file types.

    3. application/controllers/admin/Manage_cms.php

      This controller controls all the CMS Pages Management functionalities. All methods of Manage_cms class and their functionality are given below.

      • __construct(): Loads the required library, model, and helper. Also, assign some variables for using in the other methods.
      • index(): The CMS pages list with add, edit, delete, and details link.
      • add(): Add new CMS page information.
      • edit(): Update CMS page content.
      • details(): CMS page details.
      • delete(): Delete page content.
      • block(): Update page status to inactive.
      • unblock(): Update page status to active.
      • title_check(): Validation callback function, used to check existing page title.

    4. application/controllers/admin/Site_settings.php

      This controller controls all the Site Settings Management functionalities. All methods of Site_settings class and their functionality are given below.

      • __construct(): Loads the required library, model, and helper. Also, assign some variables for using in the other methods.
      • index(): Point to the settings update method.
      • edit(): Update site settings.

    5. application/controllers/Home.php

      This controller controls all the homepage management functionalities. All methods of Home class and their functionality are given below.

      • __construct(): Loads the required library, model, and helper. Also, assign some variables for using the other methods.
      • index(): Loads homepage of the frontend.

  3. application/helpers/

    1. application/helpers/common_helper.php

      The Common helper is used to define some most used helper functions. This helper file contains the following functions.

      • excerpt(): Creates excerpt of the large content.

    2. application/helpers/upload_helper.php

      The Upload helper is used to upload the file and create the thumbnail of images. This helper file contains the following functions.

      • upload(): Uploads file to the server and return the uploaded file information.
      • create_thumb(): Creates the thumbnail of the image file.

    3. application/helpers/user_email_helper.php

      This helper file is used to send an email to the admin on reset password. This helper file contains the following functions.

      • forgotPassEmail(): Send reset password link to the admin.

  4. application/libraries/

    1. application/libraries/Admin_init_elements.php

      Admin_init_elements class separates view by elements for admin panel. The following functions are used to create elements.

      • init_elements(): Initialize the element functions.
      • is_admin_loggedin(): Check the admin login status.
      • init_head(): Load head element view.
      • init_header(): Load header element view.
      • init_navigation(): Load navigation panel element view.
      • init_footer(): Load footer element view.

    2. application/libraries/Public_init_elements.php

      Public_init_elements class separates view by elements. The following functions are used to create elements.

      • init_elements(): Initialize the element functions.
      • init_head(): Load head element view.
      • init_header(): Load header element view.
      • init_footer(): Load footer element view.

  5. application/models/

    1. application/models/Admin_user.php

      The Admin_user model controls all the database related works for admin, like fetch, insert, and update data to the database.

      • getRows(): Fetch admin account data from the database based on the given conditions.
      • insert(): Insert admin account data to the database.
      • update(): Update admin account data to the database.
      • status_update(): Update admin account status.
      • delete(): Delete admin account from the database.
      • fieldValueCheck(): Check admin user data by fields and values.
      • loginCheck(): Existing admin user check.

    2. application/models/User.php

      The User model controls all the database related works for users, like fetch, insert, and update data to the database.

      • getRows(): Fetch user data from the database based on the given conditions.
      • insert(): Insert user data in the database.
      • update(): Update user data in the database.
      • delete(): Delete user account from the database.

    3. application/models/Cms.php

      The Cms model controls all the database related works for cms pages, like fetch, insert, and update data to the database.

      • getRows(): Fetch page data from the database based on the given conditions.
      • insert(): Insert page data in the database.
      • update(): Update page data in the database.
      • delete(): Delete page content from the database.

    4. application/models/Settings.php

      The Settings model controls all the database related works for site settings, like fetch, insert, and update data to the database.

      • getRow(): Fetch settings data from the database.
      • update(): Insert or Update site settings in the database.

        This view contains users list HTML and loaded by the index() method of Manage_users controller.

      2. application/views/admin/manage_users/details.php

        This view contains user details HTML and loaded by the details() method of Manage_users controller.

      3. application/views/admin/manage_users/add-edit.php

        This view contains user add/edit form HTML and loaded by the add()/edit() methods of Manage_users controller.

    6. application/views/admin/manage_cms/

      1. application/views/admin/manage_cms/index.php

        This view contains cms pages list HTML and loaded by the index() method of Manage_cms controller.

      2. application/views/admin/manage_cms/details.php

        This view contains cms page details HTML and loaded by the details() method of Manage_cms controller.

      3. application/views/admin/manage_cms/add-edit.php

        This view contains cms page add/edit form HTML and loaded by the add()/edit() methods of Manage_cms controller.

    7. application/views/admin/site_settings/

      1. application/views/admin/site_settings/edit.php

        This view contains settings edit form HTML and loaded by the edit() methods of Site_settings controller.

    8. application/views/layout.php

      This file is main application layout for front-end, it merges all the elements including the main dynamic content of the webpage.

    9. application/views/elements/

      1. application/views/elements/head.php

        The entire HTML content which will go under the <head> tags is placed here.

      2. application/views/elements/header.php

        The entire HTML content of header section is placed here.

      3. application/views/elements/footer.php

    10. application/views/home/

      1. application/views/home/index.php

        This view contains homepage HTML for the frontend.
